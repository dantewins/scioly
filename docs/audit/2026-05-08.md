### Track A — Slow (4)

- (f38) `app/api/admin/events/[id]/enrollments/route.ts:18-23` — `memberSeason: { include: { user: { select: ... } } }` ships every text-heavy column on `MemberSeason` (`whyJoin`, `contributionIdeas`, `awards`, `previousEvents`, `scienceClasses`, `mathClasses`, `questions`, `notes`, `statusReason`) for every enrolled member. Same pattern as the f30–f35 family. Inner `include` should be `select { id, user: { select: ... } }`. — static
- (f39) `app/api/member/hours/route.ts:15-19` — `prisma.hourEntry.findMany` returns every entry the member ever submitted with no `take` limit and no pagination. A long-tenured member could have hundreds of rows; ship grows linearly with seasons. Add a `take` cap (e.g. 100) consistent with `app/api/member/hub/route.ts:192-204` which already takes 10 most recent. — static
- (f40) `app/dashboard/club-events/page.tsx:16-29` — `clubEvent.findMany` (line 16) and `hourCategory.findMany` (line 25) are independent but `await`-ed sequentially; should be `Promise.all`. — static

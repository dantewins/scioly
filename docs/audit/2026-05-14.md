## Findings (51 new)

### P1 — Broken (10)

- [ ] (f78) **Member dashboard is admin-only — links 307-redirect members back to /dashboard.** `app/dashboard/page.tsx` renders MetricCards "Active Members", "Pending Applications", "Hours Awaiting Review" and links them to admin routes the member can't access. Member lands on a useless page and any link click loops. Hub endpoint (`/api/member/hub`) already returns everything needed for a personalized dashboard (announcements, assignments, hours, forms, finances). — member walk
- [ ] (f79) **Settings is hard-blocked for members — no profile surface anywhere.** `app/dashboard/settings/page.tsx:21` redirects members to `/dashboard`. There is no page to change name, email, password, graduation year, or view membership status. — member walk
- [ ] (f80) **Finances page admin-only — members can't see what they owe.** `app/dashboard/finances/page.tsx:17` requires `view_finances`; `memberPermissions()` doesn't grant it. Hub returns `finances.openInvoiceCount`/`outstandingBalanceCents` but no UI consumes them for members. — member walk
- [ ] (f81) **Competition detail page renders admin roster manager (27KB) to members in read-only mode.** `app/dashboard/competitions/[id]/page.tsx:140` always renders `<CompetitionRosterManager>` with `canManage={false}`. Member should see "you're on Team A, your events, your time slots", not a disabled admin UI. — member walk
- [ ] (f82) **Member form upload still asks members to paste a URL** — `features/forms/components/member-forms-view.tsx:133-141` uses a text input with copy "Paste a link to your uploaded document" even though `app/api/member/forms/route.ts:36,56-72` accepts a `fileAssetId` from the real upload pipeline. The asset flow exists and works (used by hour proofs); forms UI was missed. — member walk
- [ ] (f83) **Board members can't post announcements.** `app/dashboard/page.tsx:65` gates `AnnouncementComposer` on `edit_club_settings`. `boardMemberPermissions()` (`lib/permissions.ts:48-60`) explicitly omits all `club_settings` flags. Either move announcements to their own permission area or accept a less-privileged flag. — owner walk
- [ ] (f84) **Reject hours allows empty reason — API silently substitutes `"No reason provided."`** `app/api/admin/hours/route.ts:69`. The reject dialog accepts an empty textarea and submits. Either require client-side or display the literal default on the rejected card so the member sees what happened. — owner walk
- [ ] (f85) **`view_club_settings` is unreachable through the roles UI.** `features/settings/components/roles-manager.tsx:166-186` exposes only `view_roles`, `edit_roles`, `edit_club_settings`. No role can be granted view-only club_settings access; the permission is dead. — owner walk
- [ ] (f86) **Settings link hidden from board members in sidebar** (`components/app-sidebar.tsx:196` — requires `canView("club_settings")`). Compounds with f85: they can't even view roles they're allowed to edit through other paths. — owner walk
- [ ] (f87) **Roster `division` Select mismatched against API enum** — observed value `"DIV_C"` submitted by the create dialog while `app/api/admin/competitions/[id]/rosters/route.ts:14` accepts `"B"|"C"|"OTHER"`. Form sends the Prisma label, API expects the short value. — owner walk

## Wave summary

**Wave 1 (perf):** `lib/auth.ts` `loadCurrentUser` split into a fast OWNER/APPLICANT bypass that no longer JOINs `seasonMemberships`. Members/admins do a focused second query only when they actually need season-role permissions. `lib/db.ts` `getActiveSeason` now also selects `name` (was id-only). Big wins on the WEBSITE_OWNER routes that regressed in this morning's run.

**Wave 2 (member-facing P1s):**
- `app/dashboard/page.tsx`: branches by role. Non-admins (no `view_members`/`view_finances`/`view_roles`) see a new `<MemberDashboard>` fed by `features/dashboard/lib/load-member-dashboard.ts` — personalized greeting, 4 metrics (hours / dues / forms / competitions), hour progress bar, your-upcoming-competitions, continue-practice, recommended-practice, recent-results-with-color-coded-percent, upcoming-club-events, announcements. (f78, f98)

## Findings (29 new + 24 carry-over)

### P1 — Broken (5 new)

- [ ] (f44) **SECURITY: members can read practice answer keys** — `app/api/admin/practice/[id]/answer-key/route.ts:10-18` is gated on `view_practice`, which the seeded Member role grants (`prisma/migrations/20260407123000_backend_audit_cleanup/migration.sql:16`). `GET /api/admin/practice/<id>/answer-key` returns the full answer array to any logged-in member. Same shape: `GET /api/admin/practice` and `GET /api/admin/practice/[id]` ship `answerKey` + `answerKeyText` on every prompt to members; `GET /api/admin/competitions` returns unpublished competitions (e.g. `"Test Division Competition" isPublished:false`) to members. Fix: gate read endpoints that ship answer-key/admin-only fields on `edit_practice`, or strip those fields in member-visible responses, and add a dedicated `view_practice_admin` permission. — member walk (live)
- [ ] (f45) **Practice attempt status never moves to `SCORED`** — `lib/practice-assessments.ts:1086-1094` always writes `status: "SUBMITTED"` on submit, even when `scoreEarned`/`scorePossible` are non-null. All consumers (`attempt-workspace.tsx:146`, `[assessmentId]/page.tsx:130`, `practice-feed.tsx:104`) gate score display on `attempt.status === "SCORED"`, so the score row never renders. Fix: set status to `SCORED` whenever `scorePossible !== null`. — member walk (live)
- [ ] (f46) **Member hour entries are self-approved** — `app/api/member/hours/route.ts:60` sets `approvedById: user.id` on auto-approve. Audit log says "Casey Lee approved Casey Lee's hours." Fix: leave `approvedById: null` for auto-approval, or store a `system` sentinel. — member walk (live)
- [ ] (f54) **Reject path drops reviewer audit trail** — `app/api/admin/hours/route.ts:58-64` `action: "reject"` writes only `status` + `rejectionReason`; never records `approvedAt`/`approvedById` (or any `reviewedAt`/`reviewedById` analog). After rejection there is zero record of which admin made the call. Schema has no reviewer column for rejections at all — needs `reviewedById` + `reviewedAt` added or reuse `approvedById`/`approvedAt` semantics. — owner walk (live)
- [ ] (f55) **Member-side hour SSR is unbounded** — `app/dashboard/hours/page.tsx:64-68` `prisma.hourEntry.findMany` has no `take`. f39 already capped the `/api/member/hours` API path; the SSR path was missed. A long-tenured member's first paint ships every hour entry. — owner walk (static)

### Track A — Engineering health

#### Slow with empirical backing (3 new)

- [ ] (f56) `app/api/admin/members/route.ts:37-49` — `prisma.memberSeason.findMany` `include`s every text-heavy column on MemberSeason (`whyJoin`, `contributionIdeas`, `awards`, `previousEvents`, `scienceClasses`, `mathClasses`, `questions`, `notes`, `statusReason`, `focusPageFileUrl`, `expectedHours`, …). Same family as f30–f35 / f38; tighten the inner `include` to a `select`. Drives `/dashboard/members` (330ms / +26%). — owner walk (static)
- [ ] (f57) `lib/practice-assessments.ts:1097-1099` — `upsertMemberPracticeAttemptResponses` returns the full attempt detail (all prompts, parts, assets, event, PDFs) on every autosave PATCH. - [ ] (f58) `app/dashboard/competitions/page.tsx:40-47` — member-side query lists **every** season-wide competition (including `isPublished: false`). Member should see only competitions where they have a roster assignment (or only published ones). Drives `/dashboard/competitions` MEMBER (343ms, +73%) and ships unpublished admin records over the wire. — member walk (live)

**Wave 3 (Track B demo polish):**
- Competition publish flow: `notes` + `isPublished` switch on create dialog, `CompetitionPublishToggle` on detail page, draft/published badge on list, `view_competitions` → `edit_competitions` on admin GET.

- Hours history: admin queue now fetches all entries (`take: 500`) with `approvedBy`/`rejectedBy` includes, view splits into Pending/Approved/Rejected/All filter buttons; member side adds a status filter chip row.

**Wave 4 (ontology + seed):**
- `MemberSeason` ↔ `MembershipApplication` dedup: apply route no longer double-writes the application-form fields (whyJoin, contributionIdeas, etc.); reader in `members/[id]/page.tsx` no longer falls back to MemberSeason copies. Schema columns left in place for backwards-compat; safe to drop in a follow-up migration.
- f67 seed PDFs: `public/sample-packet.pdf` (minimal valid one-page PDF) added; `seed-large.ts` now creates a `SOURCE_PDF` AssessmentAsset; existing rows backfilled via `prisma db execute`. Attempt workspace now renders the source PDF in an inline iframe (or "ask your admin" placeholder if none).

**Wave 5 (model wiring — Tier 2):**
- `Announcement` wired: `app/api/admin/announcements/route.ts` (GET for members, POST gated on `edit_club_settings`), `AnnouncementComposer` dialog, hub renders pinned + recent announcements on `/dashboard`.
- `ActivityLog` wired: `lib/activity.ts` helper, logs `Competition.publish`/`unpublish` on PATCH, `HourEntry.approve`/`reject` on admin review. Best-effort; failures don't break the underlying mutation.
